# Vigit Review

Resolve the comments generated by Vigit for the current Git worktree.

## Read the handoff

1. Read repository instructions.
2. Run `git rev-parse --show-toplevel`.
3. Open the absolute `comments.md` path supplied in the prompt.
4. Confirm its `Worktree` value is the exact current root.
5. Stop before changing files when the path is missing, belongs to another
   worktree, or contains no comments.

## Resolve comments

Process every `VIGIT-*` heading in file order:

1. Inspect the saved source range and diff context.
2. Re-read the current project file; treat the anchor as possibly stale.
3. Make the smallest safe code change that addresses the comment.
4. Use the project's documented verification commands when available.
5. Preserve unrelated user and agent changes.

Do not edit `comments.md`. Do not stage, commit, push, switch branches, or use
another worktree. If a comment is ambiguous, stale, unsafe, or blocked, leave
the code unchanged for that item and explain the blocker.

## Report

Return a short list with:

- resolved comment IDs and changed files;
- blocked comment IDs and reasons;
- verification commands and results.

The user reviews the diff in Vigit and deletes completed comments manually.
